# Security Policy

## 1. 报告方式 / Reporting

中文：
如果你发现了安全问题，请不要先公开提交 issue。公开发布后，建议优先使用 GitHub 的 private vulnerability reporting。

English:
If you discover a security issue, please do not open a public issue first. Once the repository is public, GitHub private vulnerability reporting should be the preferred path.

## 2. 建议包含的信息 / Suggested Report Content

中文：
- 影响范围
- 复现步骤
- 风险判断
- 受影响命令、文件或流程

English:
- Impact scope
- Reproduction steps
- Risk assessment
- Affected commands, files, or workflows
