# Contributing

## 1. 如何参与 / How To Contribute

中文：
欢迎通过 issue、讨论、文档改进、示例补充、测试补充和代码提交来参与。

English:
Contributions are welcome through issues, discussions, documentation improvements, examples, tests, and code changes.

## 2. 开始之前 / Before You Start

中文：
- 先阅读 `README.md`
- 先确认是否已有相同 issue 或提案
- 大改动请先开 issue 或 discussion 对齐方向

English:
- Read `README.md` first
- Check whether a similar issue or proposal already exists
- For larger changes, open an issue or discussion first to align on direction

## 3. 建议流程 / Suggested Flow

中文：
1. fork 或创建分支
2. 做尽量小而清晰的改动
3. 补必要测试或最小验证
4. 提交 PR，并说明目的、范围和验证方式

English:
1. Fork the repo or create a branch
2. Make the change as small and clear as possible
3. Add the necessary tests or the smallest valid verification
4. Open a PR with the goal, scope, and validation method

## 4. PR 期待 / PR Expectations

中文：
- 改动说明清楚
- 关联相关 issue
- 命令、文档和示例同步更新
- 不引入无关重构

English:
- The change is clearly explained
- Related issues are linked
- Commands, docs, and examples are updated together
- The PR avoids unrelated refactors

## 5. 行为约定 / Collaboration Expectations

中文：
请保持友善、具体、可复现。对问题本身直接，对人保持尊重。

English:
Please be kind, specific, and reproducible. Be direct about the problem, and respectful toward people.
